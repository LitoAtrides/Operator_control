# Техническое задание v1.0  
## Desktop-приложение системы электронной очереди

---

## 1. Общая информация

**Назначение:**  
Desktop-приложение для отображения и работы с веб-интерфейсами системы электронной очереди.

**Платформы:**  
- Windows (основная)  
- macOS (дополнительно)

**Язык разработки:** Python 3.10+

**UI-фреймворк:** PySide6 (Qt)

**Браузерный движок:** Chromium (Qt WebEngine)

**Тип приложения:** Desktop, standalone (без установленного Python)

---

## 2. Цель приложения

Приложение должно:
- открывать веб-страницы системы электронной очереди
- формировать URL динамически на основе конфигурации
- поддерживать три режима работы:
  1. Табло вызова
  2. Информационная панель
  3. Пульт оператора
- в режиме **Пульта оператора** отображать **несколько пультов одновременно на одном экране**

---

## 3. Режимы работы

### 3.1 Табло вызова (Display)

**Назначение:**  
Отображение очереди пациентов (табло).

**Поведение:**
- отображается **одна веб-страница**
- одно окно
- один Chromium-браузер

**URL:**
```
/?app=equeuedisplay&platform=all
```

---

### 3.2 Информационная панель (Info)

**Назначение:**  
Отображение информационной панели и карты.

**Поведение:**
- отображается **одна веб-страница**
- одно окно
- один Chromium-браузер

**URL:**
```
/?app=infopanel
&widget=map-infopanel-tv
&panel=1
&segment={segment}
```

---

### 3.3 Пульт оператора (ORC)

**Назначение:**  
Работа операторов с очередью.

### Ключевое требование

В режиме ORC приложение должно **одновременно отображать несколько пультов оператора в одном окне**, аналогично планшетному приложению.

#### Поведение:
- одно окно приложения
- **несколько встроенных браузеров (Chromium)**
- каждый браузер соответствует **своему `orcNumber`**
- все пульты **видны одновременно**
- не использовать вкладки, переключение страниц или отдельные окна

#### Расположение:
- вертикально друг под другом
- при нехватке высоты используется вертикальная прокрутка

#### URL для каждого пульта:
```
/?app=equeueorc&platform=all&orcnumber={orcNumber}
```
